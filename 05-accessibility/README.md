# HTML Accessibility 

Ниже приведён набор вопросов и кратких ответов по теме «Доступность (Accessibility, a11y) в HTML» — важные концепты, теги, атрибуты и best practices для создания доступных веб-интерфейсов.

---

## Вопросы:

<details>
  <summary>1. Что такое доступность в контексте HTML?</summary>
  
  Доступность -  это создание веб-интерфейса, пригодного для использования людьми с инвалидностью, в том числе с нарушениями зрения, двигательных функций, слуха или когнитивных способностей.

</details>

<details>
  <summary>2. Зачем нужна доступность веб-страниц?</summary>
  
  Доступность гарантирует, что все пользователи могут взаимодействовать с сайтом. Это важно для соблюдения законодательства, расширения аудитории, улучшения UX и SEO.
</details>

<details>
  <summary>3. Какие HTML‑теги и атрибуты помогают сделать страницу доступной?</summary>

  Семантические теги (например, `<nav>`, `<main>`, `<header>`, `<footer>`, `<section>`, `<article>`), атрибуты `alt` (для изображений), `label`, `tabindex`, `aria-*`, `role`.

</details>

<details>
  <summary>4. Для чего нужен атрибут alt у изображений?</summary>
  
  `alt` даёт текстовое описание для screen reader-ов и отображается, если изображение не загружено. Для декоративных изображений — `alt=""`.
</details>

<details>
  <summary>5. Как работает tabindex и зачем его использовать?</summary>
  
  `tabindex` управляет порядком обхода элементов через клавишу Tab; `tabindex="0"` — в стандартном порядке, `-1` — программно, положительные значения — ручной порядок (используйте осторожно).
</details>

<details>
  <summary>6. Что такое aria‑атрибуты и зачем они нужны?</summary>
  
  `aria-*` (Accessible Rich Internet Applications) атрибуты добавляют описания, роли, состояния и свойства компонентам, чтобы screen reader-ы корректно их интерпретировали.
</details>

<details>
  <summary>7. Как правильно использовать role?</summary>
  
  Атрибут `role` описывает назначение элемента для вспомогательных технологий (например, `role="button"`, `role="banner"`, `role="navigation"`).
</details>

<details>
  <summary>8. Почему важно использовать &lt;label&gt; для полей ввода?</summary>

  `<label>` связывает описание/метку с конкретным инпутом, делает форму доступной для screen reader-ов и облегчает навигацию.

</details>

<details>
  <summary>9. Какие ошибки часто допускают при разметке для доступности?</summary>

  - Пропускают `alt` у изображений  
  - Используют несемантические теги для важных областей  
  - Нет специальных меток (`<label>`, `aria-label`)  
  - Не тестируют клавиатурное управление  
  - Неправильный порядок Tab

</details>

<details>
  <summary>10. Как тестировать доступность сайта?</summary>
  
  Используйте автоматические инструменты (Lighthouse, axe, WAVE), screen reader-ы (NVDA, VoiceOver), проверяйте tab-порядок и цветовой контраст вручную.

</details>

<details>
  <summary>11. Что такое landmark‑теги? Примеры применения.</summary>

  Это структурные области: `<main>`, `<nav>`, `<header>`, `<footer>`, `<aside>` — позволяют screen reader-у быстро ориентироваться на странице.
</details>

<details>
  <summary>12. Как обеспечить доступность динамического контента (SPA, модальные окна)?</summary>
  
  Для динамики используйте aria-атрибуты (`aria-live`, `aria-modal`), управляйте фокусом (`tabindex`, js), задавайте явные роли.
</details>

<details>
  <summary>13. Как улучшить доступность форм?</summary>

  - Всегда используйте `<label>`  
  - Сопоставляйте `for="id"`  
  - Добавляйте aria-атрибуты для ошибок  
  - Используйте понятные placeholder и чёткие сообщения об ошибке  
  - Проверяйте tab-порядок и визуальные индикаторы фокуса

</details>

<details>
  <summary>14. Какой цветовой контраст рекомендуется для текста и фона?</summary>
  
  WCAG рекомендует контраст не ниже 4.5:1 для обычного текста и 3:1 — для крупного/жирного.

</details>

<details>
  <summary>15. Какие ещё best practices по доступности?</summary>
  
  - Семантическая HTML-структура  
  - Явные текстовые метки  
  - Доступность с клавиатуры  
  - Screen reader-friendly компоненты  
  - Своевременные aria-аннотации  
  - Проверка на разных устройствах и браузерах
</details>
