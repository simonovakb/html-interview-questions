# Media

Краткие вопросы и ответы по теме «Медиа в HTML»: как работать с изображениями и видео, оптимизация, адаптивность, современные форматы, производительность.

---

## Вопросы:

<details>
  <summary>1. Какие теги используются для вставки изображений и видео?</summary>

  - `<img>`, `<picture>` - для изображений
  - `<video>` - для видео
  - `<audio>` - для аудио
  - `<source>` применяется внутри `<picture>`, `<video>`, `<audio>` для указания альтернативных форматов.

</details>

<details>
  <summary>2. Для чего нужен атрибут alt у изображений?</summary>
  
  `alt` — текстовое описание картинки. Важно для доступности, SEO и отображается при ошибке загрузки изображения.
</details>

<details>
  <summary>3. Как реализовать адаптивные изображения?</summary>

  С помощью атрибутов `srcset`, `sizes` у `<img>`, а также через контейнер `<picture>` со множеством `<source>` для разных форматов/размеров.

</details>

<details>
  <summary>4. Для чего нужен &lt;picture&gt;?</summary>

  `<picture>` позволяет выбирать формат и изображение в зависимости от устройства/браузера. 
  Применяют для WebP, AVIF, Retina и адаптивного дизайна.

</details>

<details>
  <summary>5. Какие современные форматы изображений оптимальны для web?</summary>
  
  - WebP, AVIF, SVG. Они дают лучшую компрессию и качество при меньшем размере. 
  - JPEG и PNG — для совместимости
  - SVG — для векторной графики.

</details>

<details>
  <summary>6. Как оптимизировать загрузку медиа?</summary>

  Используйте атрибуты `loading="lazy"` для `<img>`/`<iframe>`, применяйте современные форматы, минимизируйте размер/разрешение, используйте CDN и кэширование.

</details>

<details>
  <summary>7. Как работает тег &lt;video&gt; и его важные атрибуты?</summary>

  `<video>` — встроенное видео. 
  
  Атрибуты: `src`, `controls`, `autoplay`, `muted`, `loop`, `poster`, `<source>` для форматов.

</details>

<details>
  <summary>8. Что такое прогрессивная загрузка изображений?</summary>
  
  Загрузка сначала уменьшенной (preview) версии, а затем — полноразмерной. Быстрее выдаёт первый "видимый" результат.
</details>

<details>
  <summary>9. Какие best practices по оптимизации медиа ресурсов?</summary>

  - Использовать современные форматы (WebP, AVIF)
  - `<img srcset>`/`<picture>`
  - Ленивая загрузка (`loading="lazy"`)
  - Минифицировать размер файла
  - CDN/кэширование
  - Указывать размеры (`width`, `height`) для избежания layout shift

</details>

<details>
  <summary>10. Зачем указывать width и height у изображений?</summary>
  
  Это резервирует место (размер) ещё до загрузки, предотвращает сдвиги контента (Layout Shift), улучшает CLS метрику производительности.
</details>

<details>
  <summary>11. Как добавить субтитры и описания для видео?</summary>

  Используйте вложенный тег `<track kind="subtitles" srclang="..." src="...">` внутри `<video>`. Атрибуты — для языков, описания, субтитров.

</details>

<details>
  <summary>12. Какие атрибуты важны для accessibility медиа?</summary>

  - Для изображений — `alt`, для видео — субтитры (`<track>`), транскрипции
  - Доступные элементы управления (`controls`)
  - Разумные подписи и описания

</details>

<details>
  <summary>13. Как работать с SVG?</summary>

  SVG — векторные изображения, можно вставлять через `<img>`, `<object>` или инлайн-код `<svg>...</svg>`. У SVG доступен полный контроль над стилями и структурами через CSS и JS.

</details>

<details>
  <summary>14. Как обеспечить совместимость медиа на разных устройствах и браузерах?</summary>

  Используйте `<picture>`/`<source>` с fallback, предлагайте альтернативные форматы, проверяйте поддержку через Can I Use и тестируйте на ключевых устройствах.

</details>

<details>
  <summary>15. Какие ошибки часто встречаются при работе с медиа?</summary>
  
  - Нет alt для изображений
  - Большие (несжатые) картинки/видео
  - Игнорирование современных форматов
  - Не указаны размеры width/height
  - Нет ленивой загрузки
  - Недоступные элементы управления для видео/аудио
</details>
