# Performance

Краткие вопросы и ответы по теме «Производительность в HTML»: загрузка ресурсов, оптимизация, критический рендеринг, фишки для ускорения сайта и лучшие практики.

---

## Вопросы:

<details>
  <summary>1. Какие HTML-фичи влияют на производительность страницы?</summary>

  - расположение/загрузка скриптов
  - оптимизация медиа
  - ленивый рендеринг
  - минимизация DOM
  - использование preload/preconnect
  - критический CSS и современные форматы.

</details>


<details>
  <summary>2. Как влияют `defer` и `async` на загрузку JavaScript?</summary>

  - `defer` — скрипты загружаются параллельно, выполняются после парсинга HTML.
  - `async` — загружаются параллельно и сразу выполняются (может прервать парсинг).

</details>

<details>
  <summary>3. Зачем выносить стили во внешний файл?</summary>
  
  Внешние стили эффективнее кэшируются браузером, ускоряют повторные загрузки, разделяют логику от представления.
</details>

<details>
  <summary>4. Что такое критический CSS?</summary>
  
  Критический CSS — стили, необходимые для первого видимого рендера. Их встраивают инлайн или быстро загружают, чтобы ускорить отрисовку above the fold.
</details>

<details>
  <summary>5. Как работает `&lt;link rel="preload"&gt;` и `preconnect`?</summary>

  - `preload` - заранее загружает критичные ресурсы (шрифты, скрипты, стили).
  - `preconnect` - устанавливает соединение с доменом до реальной загрузки (ускоряет работу CDN/API).

</details>

<details>
  <summary>6. Какие best practices по оптимизации изображений?</summary>

  - Использовать `srcset`/`picture` для адаптива
  - современные форматы (WebP, AVIF)
  - `loading="lazy"`
  - оптимальные размеры
  - CDN
  - кэш
  - указывать размеры `width`/`height`.

</details>

<details>
  <summary>7. Как минимизация и сжатие ресурсов влияет на скорость?</summary>
  
  Сжатые/минифицированные HTML, CSS и JS уменьшают объем передаваемых данных, ускоряют загрузку и рендер страницу.
</details>

<details>
  <summary>8. Что такое Critical Rendering Path и как его оптимизировать?</summary>

  Это цепочка операций браузера от скачивания HTML до отрисовки страницы: парсинг HTML, CSSOM, построение DOM, layout, paint. 
  
  Оптимизация: минимизировать render-blocking ресурсы, инлайнить критический CSS, использовать `defer`.

</details>

<details>
  <summary>9. Как работает ленивая загрузка?</summary>
  
  `loading="lazy"` для `&lt;img&gt;` и `&lt;iframe&gt;` — загружает объект только когда он появляется в viewport. Экономит трафик и ускоряет первый render.
</details>

<details>
  <summary>10. Какие ошибки часто встречаются в производительности HTML?</summary>

  - Не оптимизированы картинки/видео (большие, неподходящие форматы)
  - Много `&lt;script&gt;` в head без defer/async
  - Нет preload/preconnect для критичных ресурсов
  - Большой, сложный DOM
  - Render‑blocking CSS/JS
  - Отсутствие кэширования

</details>


<details>
  <summary>11. Как использовать инструменты для анализа производительности?</summary>
  
  Используйте Chrome DevTools, Lighthouse, WebPageTest, инструменты анализа загрузки, смотрите показатели TTFB, FCP, LCP, CLS.
</details>

<details>
  <summary>12. Что такое CLS и как избежать сдвига layout?</summary>
  
  CLS (Cumulative Layout Shift) — метрика сдвига макета при загрузке. Избегайте: всегда указывайте размеры медиа (`width`/`height`), не вставляйте динамический контент без резервирования области.
</details>


<details>
  <summary>13. Почему важна быстрая загрузка first paint?</summary>
  
  Чем раньше пользователь увидит первый контент (first paint, FCP/LCP) — тем выше перформанс, меньше bounce rate и выше удовлетворённость.
</details>

<details>
  <summary>14. Какие современные рекомендации по хранению и передаче ресурсов?</summary>
  
  Используйте HTTP/2, CDN, кэширование, современные форматы, минимизируйте количество запросов и их размер.
</details>
